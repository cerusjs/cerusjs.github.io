.plugins()
Source:
lib/plugins.js, Line 1
Access:
public

With this class you can manage your plugins. A plugin is an object that contains functions that are added to the cerus core. This means that with this class you can manage the plugins that will expand CerusJS' functionality.

Examples

// this is a basic example of adding a plugin
var plugin = {
  name: "example",
  version: "0.0.1",
  example: function() {
    return "example";
  }
};

cerus.plugins().add(plugin); // or cerus.use(plugin);
// -> this will add the specified plugin

cerus.example();
// -> this will now return "example"

.add(plugin)
Source:
lib/plugins.js, Line 28
Access:
public
Summary:
Adds a new plugin to the CerusJS core.

This function is used to add a new plugin to CerusJS' core. You can add the plugin using the plugin parameter. The plugin object must contain a name and a version key else it will throw a TypeError. If the plugin has already been added it will also throw an Error. If you want to re-add a plugin you have to remove the old plugin first. You can also add dependencies to your plugin. This means that before adding the plugin your list of dependencies is checked. If a dependency isn't added yet it'll throw an error. All keys, apart from the name, version and dependencies, that aren't function are removed from the object. If you add a init function (was init, but that is deprecated) that function will not be added and will be called to initialize the plugin. It is called with the cerus object as parameter. This is how a plugin should get access to the cerus object. The rest of the functions are added to the cerus core if the key isn't taken yet.

Parameters

Name
Type
Description
plugin
Object
The plugin object that contains the plugin that will be added.
plugin.name
String
The name of the plugin. (child)
plugin.version
String
The version of the plugin. (child)
plugin.dependencies
String[]
The dependencies for the plugin. (optional) (child)

Examples

var plugin = {
  name: "example",
  version: "0.0.1",
  example: function() {
    return "example";
  }
};

cerus.plugins().add(plugin);
// -> this will add the specified plugin

var plugin = {
  name: "example",
  version: "0.0.1",
  _init: function() {
    console.log("initialized the plugin");
  },
  example: function() {
    return "example";
  }
};

cerus.plugins().add(plugin);
// -> this will add the specified plugin and call the _init function.
.remove(name)
Source:
lib/plugins.js, Line 153
Access:

Summary:
Removes a plugin from the cerus core.

With this function you can remove a plugin. The plugin removed can be specified with the name parameter. A plugin cannot be removed if it hasn't been added yet. All plugins are also checked if they don't depend on the plugin that will be removed. An error will be throw if this is the case. Before the plugin is removed of the list it's function are removed from the cerus core.

Parameters

Name
Type
Description
name
String
The name of the plugin that will be removed.

Examples

// with plugin "example" already added
cerus.plugins().remove("example");
// -> will remove the cerus plugin
