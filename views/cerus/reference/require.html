.helper(name)
Source:
require.js, Line 30
Access:
public
Summary:
Loads a helper class.

This function loads helpers for you. A helper is class that contains functions that help you with a certain job. For more information about helpers: read the tutorial about it. The helper that is loaded is specified by it's name. When a helper is loaded for a second time it will load faster since it is cached. The name can also be an absolute path if you don't want to use the default routing. You can change the name of the folder where the helpers should be stored with .folders().helpers(). The path is fixed (\\ is changed to /) before it is used to ensure support for all platforms. This function can also be called directly: e.g. cerus.helper().

Parameters

Name
Type
Description
name
String
The name of the helper.

Examples

// with as project env "C:\\project\\example.js"
cerus.autoroute().helper("example");
// -> will load "C:/project/helpers/example.js"

// with as project env "C:\\project\\example.js" and using an absolute path
cerus.autoroute().helper("D:\\project\\example");
// -> will load "D:/project/example.js"

Summary:
Loads a model class.

With this function you can load a model. A model is just like MVC a class that is used for storing (and sometimes loading) data. For more information about models read the tutorial that is written about it. The model you want to load is specified with it's name. When the model is loaded for a second time it will be loaded faster since it is cached. Just like the other methods you can use an absolute path when you don't want to use the default routing. You can also change the name of the folder where the models should be stored with .folders().models(). The path is fixed (\\ is changed to /) before it is used to ensure support for all platforms. This function can also be called directly: e.g. cerus.model().
